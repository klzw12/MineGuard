# 调度服务逻辑文档

## 1. 模块概述

调度服务负责矿山运输的智能调度，包括调度计划管理、调度任务生成、车辆司机分配、路线规划、动态调整等功能。系统通过智能算法为矿山提供高效的运输调度能力。

## 2. 核心功能

### 2.1 路线模板管理
- 模板创建：创建常用运输路线模板（起点终点坐标）
- 模板查询：查询路线模板列表
- 模板更新：更新路线模板信息
- 模板删除：删除路线模板
- **说明**：路线模板仅为快捷选择用途，实际导航使用高德地图API动态规划

### 2.2 调度计划管理
- 计划创建：创建调度计划（指定起点终点坐标、计划次数、时间段、车辆、司机、路线）
- 计划查询：查询调度计划列表
- 计划更新：更新调度计划状态
- 计划取消：取消调度计划
- 任务生成：从计划生成多个调度任务

### 2.3 调度任务管理
- 任务创建：从调度计划生成调度任务
- 任务分配：分配车辆和司机
- 任务推送：推送任务给司机（通过消息服务）
- 任务接单：司机接单创建行程
- 任务取消：取消调度任务
- 任务查询：查询调度任务信息

### 2.4 智能调度
- 最佳车辆推荐：根据货物重量、车辆状态、油量等因素综合评估推荐最佳车辆
- 最佳司机推荐：根据司机评分、常用车辆匹配度智能推荐司机
- 智能分配：智能分配车辆和司机
- 动态调整：根据突发状况动态调整调度计划

### 2.5 调度规则管理
- 规则配置：配置调度规则
- 规则管理：管理和维护调度规则
- 规则执行：执行调度规则

## 3. 核心流程

### 3.1 调度计划创建流程
```
调度员创建调度计划
    ↓
指定起点终点坐标（可选路线模板）
    ↓
设置计划次数、时间段、货物重量
    ↓
系统生成调度任务（每个任务对应一次行程）
    ↓
调用车辆服务选择最佳车辆
    ↓
智能分配司机（按优先级和评分）
    ↓
推送任务给司机（通过消息服务）
```

### 3.2 智能调度流程
```
获取任务优先级
    ↓
获取可用司机列表
    ↓
按分数降序排序（过滤分数<10的司机）
    ↓
根据优先级确定候选范围：
  - 高优先级：前 1/3 可用司机
  - 普通优先级：前 2/3 可用司机
  - 低优先级：所有可用司机
    ↓
优先匹配常用车辆状态良好的司机
    ↓
选择候选范围内最高分司机
    ↓
分配车辆（优先常用车辆，否则分配最佳车辆）
    ↓
推送调度任务到司机
```

**说明**：优先级区间选择机制确保初始数据场景下不会因固定分数阈值导致无人可用。

### 3.3 司机接单流程
```
司机收到推送的任务
    ↓
查看任务详情（起点终点、时间、货物重量）
    ↓
确认接单
    ↓
系统创建行程（关联调度任务）
    ↓
开始执行行程
    ↓
使用高德地图API导航
```

### 3.4 动态调整流程

#### 车辆故障调整
```
接收车辆故障事件
    ↓
查询该车辆的所有待执行任务
    ↓
为每个任务重新分配车辆
    ↓
推送调整通知给相关司机
```

#### 司机请假调整
```
接收司机请假事件
    ↓
查询该司机的所有待执行任务
    ↓
为每个任务重新分配司机（按智能调度逻辑）
    ↓
推送调整通知给新司机
```

#### 用户请假调整（通用）
```
接收用户请假事件（userId, roleCode）
    ↓
根据角色编码确定执行人类型：
  - ROLE_DRIVER → 司机(1)
  - ROLE_REPAIRMAN → 维修员(2)
  - ROLE_SAFETY_OFFICER → 安全员(3)
    ↓
查询该用户的所有待执行任务（按executor_id和executor_type）
    ↓
根据执行人类型获取可用人员列表
    ↓
按优先级区间选择算法选择新执行人
    ↓
更新任务的executor_id（司机同时更新driver_id）
    ↓
推送调整通知给新执行人
```

#### 线路堵塞调整
```
接收线路堵塞事件
    ↓
查询受影响的待执行任务
    ↓
查找备选路线
    ↓
更新任务的起终点信息
    ↓
推送调整通知给相关司机
```

### 3.5 最佳车辆选择算法
```
查询空闲状态车辆
    ↓
根据货物重量筛选（额定载重匹配）
    ↓
计算综合得分：
  - 油量得分（油量越高越好）
  - 载重匹配得分（载重利用率70%-90%最佳）
  - 司机绑定得分（已绑定司机加分）
    ↓
按得分降序返回前5辆推荐车辆
```

## 4. 服务间调用

### 4.1 HTTP Exchange Client

| 服务 | Client | 用途 |
|-----|--------|-----|
| 用户服务 | DriverClient | 获取司机信息、可用司机列表、常用车辆 |
| 车辆服务 | VehicleClient | 获取车辆信息、最佳车辆推荐 |
| 消息服务 | MessageClient | 推送调度任务到司机 |

### 4.2 调用接口

- 调用车辆服务：
  - `GET /api/vehicle/{id}` 获取车辆信息
  - `GET /api/vehicle/{id}/status` 获取车辆状态
  - `POST /api/vehicle/best` 选择最佳车辆

- 调用用户服务：
  - `GET /api/driver/available` 获取可用司机列表
  - `GET /api/driver/{id}/common-vehicles` 获取司机常用车辆
  - `POST /api/driver/best` 选择最佳司机

- 调用消息服务：
  - `POST /api/message/send` 发送调度任务推送

## 5. 数据存储

### 5.1 MySQL存储
- route_template：路线模板表
- dispatch_plan：调度计划表
- dispatch_task：调度任务表
- dispatch_rule：调度规则表

### 5.2 数据关系
```
route_template (路线模板，可选)
    ↓ (快捷选择)
dispatch_plan (调度计划)
    ↓ (1:N)
dispatch_task (调度任务)
    ↓ (接单后 1:1)
trip (行程)
```

## 6. 状态流转

### 6.1 调度计划状态
- 待分配(0)：计划已创建，等待分配任务
- 已分配(1)：任务已分配车辆司机
- 执行中(2)：任务正在执行
- 已完成(3)：所有任务已完成
- 已取消(4)：计划已取消

### 6.2 调度任务状态
- 待接单(0)：任务已推送，等待司机接单
- 已接单(1)：司机已接单，等待开始
- 执行中(2)：任务正在执行
- 已完成(3)：任务已完成
- 已取消(4)：任务已取消

## 7. 智能调度规则

### 7.1 司机评分与优先级区间选择

| 任务优先级 | 候选司机范围 | 说明 |
|-----------|-------------|------|
| 高/紧急 | 前 1/3 可用司机 | 高分司机优先处理紧急任务 |
| 普通 | 前 2/3 可用司机 | 中等分数司机处理普通任务 |
| 低 | 所有可用司机 | 所有司机可处理 |

**核心逻辑**：
1. 获取所有可用司机，按分数降序排序
2. 过滤分数 < 10 的司机（预警阈值）
3. 根据优先级确定候选范围（区间选择）
4. 在候选范围内优先匹配常用车辆
5. 选择候选范围内最高分司机

**优势**：避免初始数据场景下因固定分数阈值导致无人可用的问题

### 7.2 常用车辆匹配

1. 优先选择司机常用车辆中状态良好的车辆
2. 常用车辆不可用时，调用车辆服务分配最佳车辆
3. 使用次数自动累加，支持设置默认车辆

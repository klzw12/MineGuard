# 调度服务逻辑文档

## 1. 模块概述
调度服务负责行程的自动生成和智能调度，包括调度任务管理、车辆和司机分配、路线优化等功能，为系统提供智能调度能力。

## 2. 核心功能

### 2.1 调度任务管理
- 任务创建：创建调度任务
- 任务更新：更新调度任务状态和信息
- 任务执行：执行调度任务
- 任务取消：取消调度任务
- 任务查询：查询调度任务信息

### 2.2 自动调度
- 自动任务生成：根据规则自动生成调度任务
- 智能分配：智能分配车辆和司机
- 路线优化：优化行驶路线
- 调度策略：根据不同场景采用不同调度策略

### 2.3 调度规则管理
- 规则配置：配置调度规则
- 规则管理：管理和维护调度规则
- 规则执行：执行调度规则

### 2.4 调度统计
- 调度数据统计：统计调度相关数据
- 调度效果分析：分析调度效果

## 3. 主要流程

### 3.1 自动调度流程
1. 读取调度规则
2. 生成调度任务
3. 智能分配车辆和司机
4. 优化行驶路线
5. 执行调度任务
6. 调用行程服务创建行程

### 3.2 任务执行流程
1. 接收任务执行请求
2. 分配车辆和司机
3. 更新任务状态
4. 调用行程服务创建行程
5. 返回执行结果

### 3.3 调度规则执行流程
1. 读取调度规则
2. 分析规则条件
3. 执行规则逻辑
4. 生成调度任务

## 4. 服务间调用
- 调用行程服务创建行程
- 调用车辆服务获取车辆信息和状态
- 调用用户服务获取司机信息
- 调用地图服务进行路线规划
- 调用AI服务获取调度建议

## 5. 数据存储
- 调度任务存储在MySQL数据库中
- 调度规则存储在MySQL数据库中
- 调度历史存储在MySQL数据库中

## 6. 技术栈
- Spring Boot
- MySQL
- MyBatis-Plus
- Scheduling
- RESTful API

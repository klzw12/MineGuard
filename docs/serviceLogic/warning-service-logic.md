# 预警服务逻辑文档

## 1. 模块概述
预警服务负责系统的预警管理，包括预警生成、预警处理、预警通知等功能，为系统提供安全监控和异常处理能力。

## 2. 核心功能

### 2.1 预警管理
- 预警生成：根据事件触发和状态异常生成预警
- 预警处理：处理和更新预警状态
- 预警查询：查询预警记录
- 预警统计：统计预警数据

### 2.2 事件触发处理
- 事件接收：接收来自其他服务的事件触发请求
- 事件分析：分析事件类型和严重程度
- 预警生成：根据事件分析结果生成预警

### 2.3 预警通知
- 多渠道通知：通过WebSocket、MQ、短信/电话等方式发送预警通知
- 通知管理：管理通知状态和历史记录

### 2.4 预警规则管理
- 规则配置：配置预警规则和阈值
- 规则管理：管理和维护预警规则

## 3. 主要流程

### 3.1 预警生成流程
1. 接收事件触发或状态异常
2. 分析事件类型和严重程度
3. 生成预警记录
4. 保存预警记录到数据库
5. 发送预警通知

### 3.2 预警处理流程
1. 接收预警处理请求
2. 更新预警状态
3. 记录处理结果
4. 保存处理记录到数据库

### 3.3 预警通知流程
1. 生成预警通知
2. 根据预警级别选择通知渠道
3. 发送通知
4. 记录通知状态

## 4. 服务间调用
- 接收行程服务的事件触发请求
- 接收车辆服务的状态异常请求
- 调用用户服务获取用户信息
- 调用MQ服务发送通知

## 5. 数据存储
- 预警记录存储在MySQL数据库中
- 预警规则存储在MySQL数据库中
- 通知记录存储在MySQL数据库中

## 6. 技术栈
- Spring Boot
- MySQL
- MyBatis-Plus
- WebSocket
- MQ

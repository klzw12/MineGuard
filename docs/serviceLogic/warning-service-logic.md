# 预警服务逻辑文档

## 1. 模块概述

预警服务负责矿山运输过程中的安全监控和预警管理，包括预警记录管理、预警规则配置、事件触发处理、轨迹分析等功能。系统通过实时监控车辆状态和司机行为，及时发现并预警潜在的安全风险。

## 2. 核心功能

### 2.1 预警记录管理
- 记录创建：系统自动检测或人工上报创建预警记录
- 记录查询：分页查询预警记录，支持按类型、级别、状态筛选
- 记录处理：处理预警记录，记录处理结果
- 记录忽略：忽略不需要处理的预警记录

### 2.2 预警规则管理
- 规则配置：配置预警触发规则和阈值
- 规则管理：管理和维护预警规则
- 规则启用/禁用：控制规则的生效状态

### 2.3 事件触发处理
- 车辆故障事件：处理车辆上报的故障信息
- 盗卸行为事件：处理超声波检测的盗卸行为
- 危险地带事件：处理车辆进入危险区域的事件

### 2.4 轨迹分析
- 通信超时检测：检测车辆心跳超时，疑似盗卸
- 轨迹偏移检测：检测轨迹剧烈偏移，疑似坠崖
- 停留超时检测：检测在安全区域长时间停留

### 2.5 预警统计
- 预警统计：统计预警总数、各级别数量、待处理数量
- 预警趋势：统计近N天的预警趋势
- 类型统计：按预警类型统计数量
- 级别统计：按预警级别统计数量

## 3. 核心流程

### 3.1 事件触发预警流程
```
车辆/司机上报事件
    ↓
接收事件触发请求
    ↓
EventTriggerProcessor处理事件
    ↓
根据事件类型设置预警类型和级别
    ↓
生成预警记录
    ↓
保存到数据库
    ↓
推送预警通知
    ↓
返回预警记录
```

### 3.2 轨迹分析预警流程
```
接收轨迹点数据
    ↓
WarningTriggerProcessor分析轨迹
    ↓
检测是否为上报事件
    ↓
是 → 处理上报事件（故障、紧急情况）
否 → 分析轨迹异常
    ↓
检测异常类型：
  - 通信超时（疑似盗卸）
  - 轨迹剧烈偏移（疑似坠崖）
  - 安全区域停留超时
    ↓
生成预警记录
    ↓
保存到数据库
    ↓
推送预警通知
```

### 3.3 预警处理流程
```
管理员/安全员查看预警列表
    ↓
选择预警记录进行处理
    ↓
填写处理结果
    ↓
更新预警状态（已解决/已忽略）
    ↓
记录处理人和处理时间
```

## 4. 预警类型说明

| 编码 | 类型名称 | 说明 | 默认级别 |
| - | - | - | - |
| 1 | 车辆故障 | 车辆上报的故障信息 | 低风险 |
| 2 | 路线偏离 | 车辆偏离规划路线 | 中风险 |
| 3 | 长时间停留 | 车辆在非指定区域长时间停留 | 中风险 |
| 4 | 危险区域 | 车辆进入危险区域 | 高风险 |
| 5 | 速度异常 | 车辆超速行驶 | 中风险 |
| 6 | 异常行为 | 盗卸等异常行为 | 高风险 |
| 7 | 疲劳驾驶 | 司机连续驾驶时间过长 | 中风险 |

## 5. 预警级别说明

| 编码 | 级别名称 | 说明 | 处理要求 |
| - | - | - | - |
| 1 | 低风险 | 需要关注，可延后处理 | 推送给维修员 |
| 2 | 中风险 | 需要及时处理 | 推送给维修员 |
| 3 | 高风险 | 需要立即处理 | 推送全体安全员、维修员 |

## 6. 预警状态说明

| 编码 | 状态名称 | 说明 |
| - | - | - |
| 0 | 待处理 | 预警已生成，等待处理 |
| 1 | 处理中 | 正在处理中 |
| 2 | 已解决 | 预警已解决 |
| 3 | 已忽略 | 预警已忽略 |

## 7. 服务间调用

- 调用车辆服务：获取车辆信息
- 调用用户服务：获取司机信息
- 调用行程服务：获取行程信息
- 调用消息服务：推送预警通知

## 8. 数据存储

### 8.1 MySQL存储
- warning_record：预警记录表
- warning_rule：预警规则表

### 8.2 Redis存储
- vehicle:track:history:{vehicleId}：车辆轨迹历史（最近10个点）
- vehicle:heartbeat:{vehicleId}：车辆心跳时间

## 9. 预警推送规则

| 预警级别 | 推送对象 | 推送方式 |
| - | - | - |
| 低风险 | 维修员 | 系统消息 |
| 中风险 | 维修员 | 系统消息 + 短信 |
| 高风险 | 全体安全员、维修员 | 系统消息 + 短信 + 语音 |

## 10. 技术栈
- Spring Boot
- MySQL
- MyBatis-Plus
- Redis（轨迹历史、心跳检测）
- WebSocket（实时推送）

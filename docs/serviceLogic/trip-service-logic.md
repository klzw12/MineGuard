# 行程服务逻辑文档

## 1. 模块概述
行程服务负责行程的管理，包括行程创建、行程状态管理、轨迹点处理等功能，为系统提供行程相关的核心业务支持。

## 2. 核心功能

### 2.1 行程管理
- 行程创建：创建新的行程
- 行程状态更新：更新行程状态（待开始、已接单、进行中、已完成、已取消）
- 行程查询：获取行程详细信息
- 行程列表：获取行程列表

### 2.2 轨迹点处理
- 轨迹点上传：接收并处理车辆轨迹点
- 轨迹点存储：存储轨迹点数据
- 轨迹点查询：查询行程轨迹
- 轨迹点分析：分析轨迹点数据，检测异常

### 2.3 事件触发处理
- 事件接收：接收事件触发请求
- 事件处理：处理不同类型的事件（车辆故障、盗泻行为、危险地带等）
- 事件响应：根据事件类型生成相应的响应

### 2.4 行程统计
- 行程数据统计：统计行程相关数据
- 行程分析：分析行程数据，提供 insights

## 3. 主要流程

### 3.1 行程创建流程
1. 接收行程创建请求
2. 校验行程信息
3. 保存行程信息到数据库
4. 返回创建结果

### 3.2 轨迹点处理流程
1. 接收轨迹点数据
2. 处理轨迹点数据
3. 保存轨迹点到数据库
4. 分析轨迹点，检测异常
5. 触发预警（如有异常）

### 3.3 事件触发处理流程
1. 接收事件触发请求
2. 处理事件，生成偏移轨迹点
3. 上传偏移轨迹点
4. 调用预警服务处理预警

## 4. 服务间调用
- 调用预警服务处理事件触发和轨迹点异常
- 调用地图服务进行路线规划
- 调用调度服务获取调度任务
- 调用统计服务统计行程数据

## 5. 数据存储
- 行程基本信息存储在MySQL数据库中
- 轨迹点数据存储在MySQL数据库中
- 行程状态数据存储在MySQL数据库中

## 6. 技术栈
- Spring Boot
- MySQL
- MyBatis-Plus
- WebSocket
- RESTful API

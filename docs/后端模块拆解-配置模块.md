# MineGuard 后端模块拆解 - 配置模块

> 本文档描述的是 **公共配置模块（`mineguard-common-*`）的职责与依赖关系**。  
> **YAML 配置拆分/加载优先级/避免错乱的规则** 统一以 `docs/后端拆解-yml-rule.md` 为准（本文不再重复）。

## 0. 重要约束（避免 yml 重复导致错乱）

- **公共模块（jar）禁止**携带 `application*.yml(yaml)` / `bootstrap*.yml(yaml)`；尤其**禁止**在 common 模块里配置 `spring.application.name`。
- 公共模块如果需要“可配置项”，应通过 `@ConfigurationProperties` 暴露，**由服务模块或 Nacos** 提供实际值。

> 说明：当前仓库中部分 common 模块已存在 `application.yml/yaml`（且包含 `spring.application.name`），这会污染所有依赖它的服务。处理建议见 `docs/后端拆解-yml-rule.md` 的“落地现状”小节。

## 1. mineguard-common-core 模块

### 1.1 模块职责

- 统一异常定义：业务异常、系统异常
- 统一响应封装：Result、PageResult
- 通用工具类：字符串、日期、JSON、加密等工具
- 常量定义：响应码、业务常量、枚举定义
- 基础数据结构：分页参数、排序参数

### 1.2 模块依赖关系

#### 模块间依赖

- **无依赖**：作为基础模块，不依赖其他业务模块
- **被依赖**：被所有其他公共模块和业务服务模块依赖

### 1.2 模块文件结构

```tree
mineguard-common-core/
├── pom.xml
└── src/main/java/com/klzw/common/core/
    ├── config/
    │   └── BaseEntityAutoFillConfig.java # 基础实体自动填充配置
    ├── constant/
    │   ├── CacheConstants.java           # 缓存常量
    │   ├── PaginationConstants.java      # 分页常量
    │   ├── RedisKeyConstants.java        # Redis键常量
    │   └── StatusConstants.java          # 状态常量
    ├── enums/
    │   ├── ResultCodeEnum.java           # 响应码枚举
    │   ├── UserStatusEnum.java           # 用户状态枚举
    │   ├── UserTypeEnum.java             # 用户类型枚举
    │   ├── VehicleStatusEnum.java        # 车辆状态枚举
    │   └── TripStatusEnum.java           # 行程状态枚举
    ├── exception/
    │   ├── BaseException.java            # 基础异常类
    │   ├── BusinessException.java        # 业务异常
    │   ├── SystemException.java          # 系统异常
    │   ├── ExceptionHandlerStrategy.java # 异常处理策略接口
    │   ├── BusinessExceptionHandlerStrategy.java # 业务异常处理策略
    │   ├── SystemExceptionHandlerStrategy.java # 系统异常处理策略
    │   ├── DefaultExceptionHandlerStrategy.java # 默认异常处理策略
    │   └── ExceptionHandlerRegistry.java # 异常处理策略注册中心
    ├── result/
    │   ├── Result.java                   # 统一响应结果
    │   └── PageResult.java               # 分页响应结果
    ├── domain/
    │   ├── BaseEntity.java               # 基础实体类
    │   └── PageRequest.java              # 分页请求参数
    ├── util/
    │   ├── StringUtils.java              # 字符串工具类
    │   ├── DateUtils.java                # 日期工具类
    │   ├── JsonUtils.java                # JSON工具类
    │   ├── EncryptUtils.java             # 加密工具类
    │   └── ValidateUtils.java            # 校验工具类
    └── resources/
```

### 1.4 核心功能说明

#### 1.4.1 异常处理机制

- **策略模式**：通过 ExceptionHandlerStrategy 接口定义异常处理策略
- **动态注册**：ExceptionHandlerRegistry 支持动态注册异常处理策略
- **优先级机制**：新注册的异常处理策略优先级更高
- **模块化异常**：支持各模块定义专属异常类和处理策略
- **错误码分区**：为每个模块分配独立的错误码范围，避免冲突
- **异常分类**：提供 BaseException、BusinessException、SystemException 等异常类

#### 1.4.2 统一响应封装

- **通用响应**：Result 类支持成功/失败状态和数据返回
- **分页响应**：PageResult 类支持分页数据的统一封装
- **链式调用**：提供便捷的静态方法进行响应构建

#### 1.4.3 通用工具类

- **字符串处理**：StringUtils 提供字符串判断、处理等方法
- **日期处理**：DateUtils 提供日期时间格式化和解析
- **JSON处理**：JsonUtils 基于 Fastjson 提供序列化和反序列化
- **加密工具**：EncryptUtils 提供 MD5、SHA-256 加密
- **数据校验**：ValidateUtils 提供手机号、邮箱、身份证等校验

#### 1.4.4 常量和枚举

- **缓存常量**：CacheConstants 定义缓存相关常量
- **分页常量**：PaginationConstants 定义分页相关常量
- **Redis键**：RedisKeyConstants 定义 Redis 键模板
- **状态常量**：StatusConstants 定义状态相关常量
- **响应码枚举**：ResultCodeEnum 定义系统响应码
- **状态枚举**：各种状态的枚举定义，如用户状态、用户类型、车辆状态、行程状态等

#### 1.4.5 基础数据结构

- **基础实体**：BaseEntity 包含通用字段如 ID、创建时间等
- **分页请求**：PageRequest 封装分页参数，支持默认值和边界检查
- **基础实体自动填充**：BaseEntityAutoFillConfig 配置基础实体字段的自动填充

## 2. mineguard-common-web 模块

### 2.1 模块职责

- 统一异常处理：全局异常捕获和处理
- 跨域配置：CORS 跨域支持
- 请求日志切面：记录请求和响应日志
- 参数校验：统一参数校验配置
- 接口文档：Swagger/Knife4j 配置
- 文件上传：文件上传配置和限制

### 2.2 模块依赖关系

#### 模块间依赖

- **依赖**：mineguard-common-core、mineguard-common-auth
- **被依赖**：所有需要提供 Web 接口的业务服务模块

### 2.3 模块文件结构

```tree
mineguard-common-web/
├── pom.xml
└── src/
    ├── main/java/com/klzw/common/web/
    │   ├── config/
    │   │   ├── WebMvcConfig.java             # Web MVC 配置（注册过滤器 + 拦截器）
    │   │   ├── CorsConfig.java               # 跨域配置
    │   │   ├── Knife4jConfig.java            # Knife4j 配置
    │   │   ├── FileUploadConfig.java         # 文件上传配置
    │   │   └── ExceptionHandlerAutoConfiguration.java # 异常处理器自动配置
    │   ├── handler/
    │   │   ├── GlobalExceptionHandler.java   # 全局异常处理器
    │   │   └── WebExceptionHandlerStrategy.java # Web 异常处理策略
    │   ├── exception/
    │   │   └── WebException.java             # Web 异常类
    │   ├── constant/
    │   │   └── WebResultCode.java            # Web 错误码枚举
    │   ├── aspect/
    │   │   ├── RequestLogAspect.java        # 请求日志切面
    │   │   └── ControllerLogAspect.java     # 控制器日志切面
    │   ├── interceptor/
    │   │   ├── RequestInterceptor.java       # 请求拦截器（与认证无关，专注日志/埋点）
    │   │   └── AuthInterceptor.java         # 认证拦截器（基于 UserContext + @IgnoreAuth）
    │   ├── filter/
    │   │   └── RequestFilter.java            # 请求过滤器
    │   └── resolver/
    │       └── CurrentUserResolver.java      # 当前用户参数解析器
    └── main/resources/
        └── （无 application*.yml；仅可选放置 Web 自身需要的消息/模版资源。  
             Web 相关 YAML 片段位于各业务服务模块的 `config/application-*.yml`，详见《后端拆解-yml-rule.md》）
```

### 2.4 核心功能说明

#### 2.4.1 全局异常处理

- **统一捕获**：GlobalExceptionHandler 使用 @RestControllerAdvice 注解统一捕获所有控制器异常
- **策略处理**：基于 ExceptionHandlerRegistry 动态选择异常处理策略
- **标准响应**：将异常转换为标准的 Result 响应格式
- **日志记录**：记录异常详细信息，便于问题排查

#### 2.4.2 Web 配置

- **跨域支持**：CorsConfig 配置跨域请求处理
- **文件上传**：FileUploadConfig 配置文件上传限制和路径
- **接口文档**：Knife4jConfig 配置 API 接口文档生成

#### 2.4.3 请求日志和监控

- **请求日志**：RequestLogAspect 记录请求和响应信息
- **控制器日志**：ControllerLogAspect 记录控制器方法执行情况
- **性能监控**：通过切面实现接口性能监控

#### 2.4.4 请求处理和拦截

- **请求拦截**：RequestInterceptor 处理请求参数和响应
- **认证拦截**：AuthInterceptor 处理用户认证信息
- **参数解析**：CurrentUserResolver 解析当前用户信息

## 12. 配置模块版本管理

所有配置模块统一版本管理，以 `backend/pom.xml` 为唯一事实来源（本文不再维护重复的版本片段，避免与实际代码漂移）。

关键原则：

- **子模块依赖不写 version**：由父 POM 的 `<dependencyManagement>` 锁定版本与坐标。
- **第三方版本统一收口**：由父 POM `<properties>` 统一管理。
- **文档版本表**：请参考 `docs/后端拆解-依赖关系和版本文档.md`（同样应以父 POM 为准）。

```xml
<!-- 以 backend/pom.xml 为准，此处仅展示写法要点 -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>com.klzw</groupId>
            <artifactId>mineguard-common-core</artifactId>
            <version>${mineguard.version}</version>
        </dependency>
        <!-- ... 其它模块同理 ... -->
    </dependencies>
</dependencyManagement>
```

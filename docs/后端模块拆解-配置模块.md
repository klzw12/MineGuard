# MineGuard 后端模块拆解 - 配置模块

> 本文档描述的是 **公共配置模块（`mineguard-common-*`）的职责与依赖关系**。  
> **YAML 配置拆分/加载优先级/避免错乱的规则** 统一以 `docs/后端拆解-yml-rule.md` 为准（本文不再重复）。

## 0. 重要约束（避免 yml 重复导致错乱）

- **公共模块（jar）禁止**携带 `application*.yml(yaml)` / `bootstrap*.yml(yaml)`；尤其**禁止**在 common 模块里配置 `spring.application.name`。
- 公共模块如果需要"可配置项"，应通过 `@ConfigurationProperties` 暴露，**由服务模块或 Nacos** 提供实际值。

## 1. mineguard-common-core 模块

### 1.1 模块职责

- 统一异常定义：基础异常、系统异常
- 统一响应封装：Result、PageResult
- 通用工具类：字符串、日期、JSON、加密等工具
- 枚举定义：响应码、用户状态、用户类型、车辆状态等
- **服务间调用Client接口**：所有 HTTP Exchange Client 接口定义
- 基础数据结构：分页参数

### 1.2 模块依赖关系

#### 模块间依赖

- **无依赖**：作为基础模块，不依赖其他业务模块
- **被依赖**：被所有其他公共模块和业务服务模块依赖

### 1.4 核心功能说明

#### 1.4.1 服务间调用机制

- **HTTP Exchange Client**：使用 Spring 6 的 `@HttpExchange` 注解定义服务间调用接口
- **统一管理**：所有 Client 接口定义在 `mineguard-common-core` 中，避免循环依赖
- **服务发现**：通过 Nacos 实现服务注册发现，Client 通过服务名调用目标服务

#### 1.4.2 统一响应封装

- **通用响应**：Result 类支持成功/失败状态和数据返回
- **分页响应**：PageResult 类支持分页数据的统一封装
- **链式调用**：提供便捷的静态方法进行响应构建

#### 1.4.3 通用工具类

- **字符串处理**：StringUtils 提供字符串判断、处理等方法
- **日期处理**：DateUtils 提供日期时间格式化和解析
- **JSON处理**：JsonUtils 提供 JSON 序列化和反序列化
- **加密工具**：EncryptUtils 提供 MD5、SHA-256 加密
- **数据校验**：ValidateUtils 提供手机号、邮箱、身份证等校验

## 12. 配置模块版本管理

所有配置模块统一版本管理，以 `backend/pom.xml` 为唯一事实来源（本文不再维护重复的版本片段，避免与实际代码漂移）。

关键原则：

- **子模块依赖不写 version**：由父 POM 的 `<dependencyManagement>` 锁定版本与坐标。
- **第三方版本统一收口**：由父 POM `<properties>` 统一管理。
- **文档版本表**：请参考 `docs/后端拆解-依赖关系和版本文档.md`（同样应以父 POM 为准）。

```xml
<!-- 以 backend/pom.xml 为准，此处仅展示写法要点 -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>com.klzw</groupId>
            <artifactId>mineguard-common-core</artifactId>
            <version>${mineguard.version}</version>
        </dependency>
        <!-- ... 其它模块同理 ... -->
    </dependencies>
</dependencyManagement>
```

# MineGuard 后端模块拆解 - 服务模块

## 1. user-service 用户服务

### 1.1 模块职责

- 用户管理：用户注册、登录、信息修改、密码重置
- 认证授权：JWT Token 生成和验证、权限控制
- 角色管理：角色定义、角色分配、权限配置
- 用户信息查询：用户详情查询、用户列表查询
- 用户状态管理：用户启用/禁用、状态变更

### 1.2 模块依赖关系

#### 模块间依赖

- **依赖**：所有公共配置模块
- **被依赖**：其他业务服务模块（通过 Feign 调用）
- **Feign 客户端**：提供用户信息查询接口供其他服务调用

## 2. vehicle-service 车辆服务

### 2.1 模块职责

- 车辆管理：车辆注册、信息修改、车辆删除
- 车辆状态监控：实时状态查询、状态变更通知
- 车辆信息查询：车辆详情查询、车辆列表查询
- 车辆保险管理：保险信息录入、保险到期提醒
- 车辆维护管理：维护记录、维护提醒
- 车辆实时位置：位置上报、位置查询

## 3. trip-service 行程服务

### 3.1 模块职责

- 行程管理：行程创建、行程修改、行程取消
- 行程记录：行程开始、行程结束、行程暂停
- 路线规划：路线计算、路线优化
- 行程查询：行程详情查询、行程列表查询
- 行程统计：行程时长统计、行驶距离统计

## 5. statistics-service 统计服务

### 5.1 模块职责

- 数据统计：车辆使用统计、行程统计、预警统计
- 报表生成：日报表、周报表、月报表
- 数据分析：趋势分析、对比分析、异常分析
- 可视化数据：图表数据生成、仪表盘数据
- 数据导出：Excel 导出、PDF 导出

## 10. 服务调用关系

### 10.1 核心调用链路

- **用户请求**：gateway-service → 对应业务服务
- **车辆管理**：user-service → vehicle-service → trip-service
- **行程管理**：user-service → trip-service → vehicle-service
- **预警处理**：vehicle-service → warning-service → user-service
- **统计分析**：statistics-service → 所有业务服务
- **成本核算**：cost-service → user-service → vehicle-service → trip-service
- **AI 分析**：ai-service → 所有业务服务（获取数据），其他服务 → ai-service（获取分析结果）
- **IoT 数据流**：设备 → iot-service → 消息队列 → 其他业务服务

## 15. 总结

本文档详细拆解了 MineGuard 后端服务的 9 个服务模块，包括：

1. **user-service**：用户服务，负责用户管理、认证授权、角色管理
2. **vehicle-service**：车辆服务，负责车辆管理、状态监控、位置管理
3. **trip-service**：行程服务，负责行程管理、路线规划、行程历史
4. **warning-service**：预警服务，负责预警管理、预警检测、预警处理
5. **statistics-service**：统计服务，负责数据统计、报表生成、数据分析
6. **cost-service**：成本服务，负责成本管理、预算管理、成本分析
7. **ai-service**：AI 分析服务，负责 AI 模型管理、预测分析、智能推荐
8. **iot-service**：IoT 服务，负责设备管理、数据采集、实时监控
9. **gateway-service**：网关服务，负责请求路由、认证授权、限流熔断

所有服务模块都遵循统一的依赖关系和版本管理策略，通过 Feign 进行服务间调用，通过 RocketMQ 进行异步消息通信，通过 WebSocket 进行实时数据推送。

详细的依赖关系和版本管理请参考《后端拆解-依赖关系和版本文档.md》。

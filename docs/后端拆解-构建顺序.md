## 4. 完整构建命令

### 4.1 公共模块构建

```bash
# 方式一：从父POM一键构建（推荐）
cd backend
mvn clean install -DskipTests

# 方式二：按层级顺序构建
# Layer 0: 父POM
mvn clean install -N

# Layer 1: 核心模块
cd mineguard-common-core && mvn clean install -DskipTests

# Layer 2: 基础功能模块（可并行）
cd ../mineguard-common-database && mvn clean install -DskipTests
cd ../mineguard-common-redis && mvn clean install -DskipTests
cd ../mineguard-common-mq && mvn clean install -DskipTests
cd ../mineguard-common-mongodb && mvn clean install -DskipTests
cd ../mineguard-common-file && mvn clean install -DskipTests
cd ../mineguard-common-map && mvn clean install -DskipTests

# Layer 3: 认证模块
cd ../mineguard-common-auth && mvn clean install -DskipTests

# Layer 4: Web模块
cd ../mineguard-common-web && mvn clean install -DskipTests

# Layer 5: WebSocket模块
cd ../mineguard-common-websocket && mvn clean install -DskipTests
```

### 4.2 服务模块构建

```bash
# 服务模块需在公共模块构建完成后进行

# Layer 1: 基础服务
cd user-service && mvn clean package -DskipTests
cd gateway-service && mvn clean package -DskipTests

# Layer 2: 核心业务服务
cd vehicle-service && mvn clean package -DskipTests
cd trip-service && mvn clean package -DskipTests
cd warning-service && mvn clean package -DskipTests

# Layer 3: 数据分析服务
cd statistics-service && mvn clean package -DskipTests
cd cost-service && mvn clean package -DskipTests

# Layer 4: AI服务
cd ai-service && mvn clean package -DskipTests

# 独立服务
cd admin-service && mvn clean package -DskipTests
```

## 5. 启动顺序

### 5.1 基础设施启动顺序

```
1. MySQL          (端口: 3306)
2. Redis          (端口: 6379)
3. MongoDB        (端口: 27017)
4. RabbitMQ       (端口: 5672, 15672)
5. MinIO          (端口: 9000, 9001)
```

### 5.2 服务启动顺序

```
第一层（基础服务）：
├── gateway-service    # API网关，入口服务
└── user-service       # 用户认证，被其他服务依赖

第二层（核心业务）：
├── vehicle-service    # 车辆管理
├── trip-service       # 行程管理
└── warning-service    # 预警管理

第三层（数据分析）：
├── statistics-service # 统计分析
└── cost-service       # 成本管理

第四层（智能服务）：
└── ai-service         # AI分析

独立服务（随时可启动）：
└── admin-service      # 后台管理
```

## 6. 注意事项

### 6.1 构建注意事项

1. **必须先构建公共模块**：服务模块依赖公共模块，必须先完成公共模块的 install
2. **层级内可并行**：同一层级的模块可以并行构建，无相互依赖
3. **跳过测试加速**：开发阶段可使用 `-DskipTests` 加速构建
4. **集成测试**：如需运行集成测试，使用 `-P integration-tests` profile

### 6.2 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 找不到 common-core | 未先构建 core 模块 | 先执行 `mvn install` 安装 core |
| auth 模块构建失败 | redis 模块未安装 | 先构建 redis 模块 |
| web 模块构建失败 | auth 模块未安装 | 先构建 auth 模块 |
| websocket 构建失败 | 多个依赖未安装 | 按顺序构建所有依赖模块 |

### 6.3 快速重建命令

```bash
# 完整重建（清理并重新构建所有模块）
cd backend
mvn clean install -DskipTests

# 仅重建指定模块及其依赖
mvn clean install -pl mineguard-common-auth -am -DskipTests

# 仅重建指定模块（不重建依赖）
mvn clean install -pl mineguard-common-auth -DskipTests
```

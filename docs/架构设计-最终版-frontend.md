# MineGuard 前端架构设计文档

## 1. 技术选型

### 1.1 核心框架

- **框架**：uni-app x
- **Vue 版本**：Vue 3
- **开发语言**：TypeScript

### 1.2 技术特性

- **响应式设计**：适配多端设备
- **组件化开发**：提高代码复用性
- **路由管理**：统一的页面导航
- **状态管理**：全局状态共享
- **网络请求**：API 接口调用
- **多入口设计**：管理端和用户端分离

## 2. 项目结构

### 2.1 整体架构

采用 **管理端和用户端分离，App 多入口的混合方案**，具体结构如下：

```txt
frontend/
├── pages/           # 页面文件
│   ├── admin/       # 管理端页面
│   │   ├── login/   # 管理端登录
│   │   ├── home/    # 管理端首页
│   │   ├── vehicle/ # 车辆管理
│   │   ├── driver/  # 司机管理
│   │   ├── monitor/ # 实时监控
│   │   ├── report/  # 统计报表
│   │   └── profile/ # 管理端个人中心
│   ├── user/        # 用户端页面
│   │   ├── login/   # 用户端登录
│   │   ├── home/    # 用户端首页
│   │   ├── task/    # 任务管理
│   │   ├── vehicle/ # 车辆状态
│   │   └── profile/ # 用户端个人中心
│   └── common/      # 通用页面
│       └── about/   # 关于我们
├── static/          # 静态资源
│   ├── avatar/      # 头像图片
│   ├── images/      # 图片资源
│   ├── logo/        # Logo
│   └── svgs/        # SVG 图标
├── components/      # 组件
│   ├── admin/       # 管理端组件
│   ├── user/        # 用户端组件
│   └── common/      # 通用组件
├── utils/           # 工具函数
├── api/             # API 接口
│   ├── admin/       # 管理端 API
│   └── user/        # 用户端 API
├── App.uvue         # 应用入口
├── main.uts         # 主入口
├── pages.json       # 页面配置
├── manifest.json    # 应用配置
└── uni.scss         # 全局样式
```

### 2.2 多入口设计

- **App 多入口**：通过 `App.uvue` 实现不同角色的入口切换
- **登录页面分离**：管理端和用户端分别有独立的登录页面
- **路由配置分离**：在 `pages.json` 中配置不同角色的路由
- **权限控制**：基于角色的权限管理

## 5. 技术实现

### 5.1 路由配置

- **pages.json**：
  - 配置管理端和用户端的页面路径
  - 管理端路由：`pages/admin/**`
  - 用户端路由：`pages/user/**`
  - 通用路由：`pages/common/**`
- **多入口配置**：
  - 管理端底部导航栏
  - 用户端底部导航栏
  - 登录页面独立配置
- **uniIdRouter**：登录页面配置，支持不同角色的登录跳转

### 5.2 状态管理

- **本地存储**：
  - 登录状态：`uni.getStorageSync / uni.setStorageSync`
  - 角色信息：存储当前用户角色
  - 权限信息：存储用户权限列表
- **组件状态**：Vue 3 ref / reactive
- **全局状态**：使用 Vue 3 provide/inject 或 Pinia 管理全局状态

### 5.3 样式设计

- **主题色**：
  - 管理端：蓝色渐变 (#007AFF → #0056b3)
  - 用户端：绿色渐变 (#34C759 → #28a745)
- **布局**：卡片式布局，响应式设计
- **响应式**：适配不同屏幕尺寸，移动端优先
- **动画**：流畅的过渡效果，提升用户体验

### 5.4 多入口实现

- **App.uvue**：
  - 应用启动时检测登录状态
  - 根据角色信息跳转到对应端的首页
  - 未登录时显示角色选择页面
- **入口页面**：
  - 管理端入口：`pages/admin/login/index.uvue`
  - 用户端入口：`pages/user/login/index.uvue`
- **权限控制**：
  - 路由守卫：控制页面访问权限
  - 组件权限：基于角色显示不同组件
  - API 权限：不同角色调用不同 API
